# Быстрый старт

## Запуск расширения в режиме разработки

Ошибка TypeScript `Cannot find module 'vscode'` - это нормально для разработки расширений VSCode. Расширение будет работать при запуске через VSCode.

### Шаг 1: Откройте расширение в VSCode

```bash
cd vscode-cn-scss-navigator
code .
```

### Шаг 2: Запустите расширение

В VSCode нажмите `F5` или:
1. Откройте меню "Run" → "Start Debugging"
2. Или нажмите `Ctrl+Shift+D` и кликните на зеленую кнопку "Run Extension"

Это откроет новое окно VSCode с заголовком "[Extension Development Host]" - в нем расширение будет активно.

### Шаг 3: Откройте ваш проект

В новом окне откройте папку с вашим проектом:
```
File → Open Folder → /home/evgender/www/arcadia/frontend/services/webmaster
```

### Шаг 4: Протестируйте расширение

1. Откройте файл `src/pages/403Page/403Page.tsx`
2. Найдите строку: `const cn403Page = cn('403Page');`
3. Наведите курсор на `'403Page'` (или на `cn403Page`)
4. Зажмите `Ctrl` (или `Cmd` на Mac) и кликните
5. Должен открыться файл `src/pages/403Page/403Page.scss`

## Альтернативный способ: Установка как локальное расширение

Если хотите установить расширение постоянно (без режима разработки):

```bash
cd vscode-cn-scss-navigator

# Создайте символическую ссылку в директорию расширений VSCode
ln -s "$(pwd)" ~/.vscode/extensions/cn-scss-navigator
```

После этого перезапустите VSCode, и расширение будет работать во всех проектах.

## Проверка работы расширения

После запуска в режиме разработки, в консоли отладки (Debug Console) вы должны увидеть сообщение:
```
CN SCSS Navigator активирован
```

Если сообщения нет, проверьте:
1. Что вы открыли TypeScript/JavaScript файл
2. Что расширение активировано (View → Extensions → CN SCSS Navigator должно быть включено)

## Устранение проблем

### Расширение не активируется

Убедитесь, что вы открыли файл с расширением `.ts`, `.tsx`, `.js` или `.jsx` - расширение активируется только для этих типов файлов.

### Навигация не работает

1. Убедитесь, что SCSS файл существует в той же директории
2. Проверьте, что имя файла совпадает с именем в `cn()` (например, для `cn('Button')` должен быть файл `Button.scss`)
3. Попробуйте также с файлами `.sass` или `.css`

### Ошибки компиляции TypeScript

Ошибка `Cannot find module 'vscode'` - это нормально. VSCode предоставляет модуль 'vscode' во время выполнения расширения. Просто игнорируйте эту ошибку и запускайте через `F5`.

## Что дальше?

После успешного тестирования вы можете:

1. **Упаковать расширение** (требуется установленный vsce):
   ```bash
   vsce package
   ```
   Это создаст файл `.vsix`, который можно установить через VSCode

2. **Настроить расширение** через настройки VSCode:
   ```json
   {
     "cnScssNavigator.scssFilePattern": "{componentName}.scss"
   }
   ```

3. **Изменить код** в `src/extension.ts` и перезапустить отладку (`Ctrl+Shift+F5`)

Подробная документация находится в [`README.md`](README.md) и [`INSTALLATION.md`](INSTALLATION.md).
